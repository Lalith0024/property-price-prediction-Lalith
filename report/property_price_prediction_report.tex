\textbf{\large Abstract}\\[6pt]
This report presents an end-to-end machine learning system for intelligent real estate
valuation and investment grade classification. The pipeline operates on a dataset of
1,800 property records, applying robust preprocessing (median imputation, ordinal and
one-hot encoding, IQR-based outlier clipping), and trains two XGBoost models: a
regressor for predicting \texttt{Current\_Market\_Price} and a multi-class classifier for
predicting \texttt{Investment\_Grade} (classes 0, 1, 2). The regression model achieves an
$R^2$ of $\mathbf{0.9483}$ with an RMSE of $\approx\!1.01\text{M}$, while the classifier
reaches $\mathbf{97.50\%}$ accuracy and a weighted F1-score of $\mathbf{0.97}$. A
production-ready Streamlit application supports both manual and batch (CSV) inference
and is publicly deployed on Streamlit Community Cloud.
\end{mdframed}

\section{Introduction}

\subsection{Motivation}

Real estate represents one of the most significant investment categories globally. Two
persistent challenges face buyers, sellers, and investors:

\begin{enumerate}
    \item \textbf{Manual valuation inconsistencies} --- property pricing varies non-linearly with
          feature interactions (location, size, amenities, market dynamics), making human
          appraisals costly and error-prone.
    \item \textbf{Investment grade ambiguity} --- classifying a property as a good investment
          requires simultaneously considering price, yield, and contextual signals that are
          difficult to synthesise manually.
\end{enumerate}

\subsection{Objective}

The \textit{Property Price Prediction} system is a production-structured machine learning
pipeline that:

\begin{enumerate}
    \item Accepts raw property attributes as input.
    \item Applies a comprehensive preprocessing and feature engineering pipeline.
    \item Predicts a continuous \texttt{Current\_Market\_Price} via XGBoost regression.
    \item Classifies a property into \texttt{Investment\_Grade} $\in \{0,\,1,\,2\}$ via XGBoost classification.
    \item Exposes predictions through an interactive, publicly deployed Streamlit application.
\end{enumerate}

\subsection{Scope}

\begin{itemize}[noitemsep]
    \item \textbf{Dataset:} 1,800 residential property records with 17 raw columns.
    \item \textbf{Tasks:} Regression (price) and multi-class classification (investment grade).
    \item \textbf{Models:} XGBoost Regressor and XGBoost Classifier.
    \item \textbf{Deployment:} Streamlit Community Cloud (public, live).
\end{itemize}

\section{Data Preprocessing Pipeline}

A structured preprocessing pipeline was implemented to prepare the raw dataset for
modelling. All steps are applied before any train/test split to prevent data leakage.

\subsection{Missing Value Imputation}

Numeric columns were imputed using \textbf{median values}, which is robust to the outliers
present in real-estate data:

\begin{lstlisting}[language=Python, caption={Median Imputation}]
num_cols = df.select_dtypes(include=np.number).columns
df[num_cols] = df[num_cols].fillna(df[num_cols].median())
\end{lstlisting}

\subsection{Categorical Encoding}

Two encoding strategies were applied depending on feature ordinality:

\subsubsection*{Ordinal Encoding --- \texttt{Furnishing\_Status}}

\begin{lstlisting}[language=Python, caption={Ordinal Encoding}]
furnish_map = {
    'Unfurnished': 0,
    'Semi-furnished': 1,
    'Fully-furnished': 2
}
df['Furnishing_Status'] = df['Furnishing_Status'].map(furnish_map)
\end{lstlisting}

\subsubsection*{One-Hot Encoding --- \texttt{Neighborhood}}

One-hot encoding with \texttt{drop\_first=True} was applied to eliminate multicollinearity
from the dummy variable trap:

\begin{lstlisting}[language=Python, caption={One-Hot Encoding}]
df = pd.get_dummies(df, columns=['Neighborhood'], drop_first=True)
\end{lstlisting}

\subsection{Outlier Treatment}

Inter-Quartile Range (IQR) clipping was applied to all numeric columns
\textit{excluding} the two target variables (\texttt{Current\_Market\_Price} and
\texttt{Investment\_Grade}) to preserve their natural distributions:

\begin{lstlisting}[language=Python, caption={IQR-Based Outlier Clipping}]
for col in num_cols:
    if col not in ['Current_Market_Price', 'Investment_Grade']:
        Q1 = df[col].quantile(0.25)
        Q3 = df[col].quantile(0.75)
        IQR = Q3 - Q1
        lower = Q1 - 1.5 * IQR
        upper = Q3 + 1.5 * IQR
        df[col] = np.clip(df[col], lower, upper)
\end{lstlisting}

The cleaned dataset was persisted as \texttt{data/processed/real\_estate\_clean.csv} for
reproducibility.

\section{Evaluation}

\subsection{Evaluation Metrics}

\subsubsection{Regression}

\begin{align}
    \text{MAE}  &= \frac{1}{n}\sum_{i=1}^{n}|y_i - \hat{y}_i| \\[6pt]
    \text{RMSE} &= \sqrt{\frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2} \\[6pt]
    R^2         &= 1 - \frac{\sum(y_i - \hat{y}_i)^2}{\sum(y_i - \bar{y})^2}
\end{align}

\subsection{Key Observations}

\begin{enumerate}
    \item \textbf{Regression:} An $R^2$ of $0.9483$ demonstrates that XGBoost explains more than
          $94\%$ of the variance in property prices. The relatively high RMSE ($\approx$1.01M)
          reflects the wide price range inherent in the real-estate domain.
    \item \textbf{Classification:} $97.5\%$ accuracy with consistently high precision and recall
          across all three investment grade classes validates the classifier's robustness.
    \item \textbf{Class 2 Recall:} The slightly lower recall for class 2 ($0.91$) compared to
          classes 0 and 1 is consistent with the lower support (22 samples), indicating a
          class imbalance concern for grade 2 properties.
    \item \textbf{Leakage Prevention:} \texttt{Current\_Market\_Price} was explicitly excluded
          from the classification feature set to prevent target leakage, ensuring the model
          generalises to cold-start predictions.
\end{enumerate}

The Streamlit inference layer (\texttt{app/streamlit\_app.py}) implements the following
function sequence:

\begin{enumerate}

    \item \texttt{build\_feature\_row(...)} --- constructs feature vector for manual input
    \item \texttt{raw\_to\_feature\_frame(...)} --- transforms raw CSV rows for batch inference
    \item \texttt{run\_predict(...)} --- scales features and calls both predict heads
    \item \texttt{page\_manual(...)} / \texttt{page\_csv(...)} --- UI page handlers
\end{enumerate}

\section{Limitations}

\begin{enumerate}
    \item \textbf{Dataset Size:} With 1,800 records, the model behaviour is tightly coupled to
          the training distribution. Out-of-distribution properties (extreme sizes, unusual
          locations) may yield less reliable predictions.
    \item \textbf{Investment Grade Semantics:} The grade labels (0, 1, 2) are numeric class IDs.
          Business meaning (e.g., Poor / Good / Excellent) should be mapped explicitly in a
          production advisory system.
    \item \textbf{Market Drift:} Real-world price dynamics change continuously. The static model
          will degrade over time without periodic retraining on fresh data.
    \item \textbf{Class Imbalance (Grade 2):} The low support for class 2 (22 test samples)
          reduces confidence in recall estimates for that grade category.
    \item \textbf{Feature Interactions:} Complex non-linear location--price interactions
          (micro-market dynamics, zoning changes) are not captured by the current feature set.
\end{enumerate}

\section{Future Work}

\subsection{Short-Term Improvements (Milestone 1+)}

\begin{itemize}[noitemsep]
    \item Apply SMOTE or class-weight balancing for Investment Grade class 2.
    \item Use SHAP values for feature importance interpretability.
    \item Expand feature engineering with location-intelligence signals (transport score,
          nearby amenity index, pincode zone index).
    \item Implement model monitoring for input and prediction drift.
\end{itemize}

\subsection{Milestone 2 --- Agentic AI Framework}

\begin{itemize}[noitemsep]
    \item \textbf{LangGraph Agent:} Autonomous reasoning over property characteristics to generate
          structured investment advisory reports.
    \item \textbf{RAG Integration:} Retrieval of real-time market trends, regulations, and
          comparable listings via Chroma/FAISS.
    \item \textbf{Structured Advisory Outputs:} Buy/Hold/Avoid recommendations with risk notes.
    \item \textbf{Explainability Layer:} Feature contribution view per prediction to justify
          price and grade estimates.
\end{itemize}

\section{Conclusion}

This project demonstrates that gradient-boosted tree models, combined with principled
feature engineering and preprocessing, can deliver high-quality automated real estate
valuation and investment grade classification. The XGBoost regression model achieves
$R^2 = 0.9483$ on held-out test data, while the XGBoost classifier reaches $97.5\%$
accuracy with strong per-class precision and recall.

The modular architecture --- separate preprocessing, model artifacts, and Streamlit
application --- enables straightforward extension toward the Milestone 2 agentic framework.
Key next steps include addressing class imbalance for grade 2 properties, incorporating
richer location features, and building the LangGraph-based advisory agent.
